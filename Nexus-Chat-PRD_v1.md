# Nexus Chat (极速社交助手) - PRD 初版

## [Project DNA]
实现高效、稳定的即时通讯与社交网络，提供流畅的消息传递体验。

## [First Action Path]
用户注册/登录 -> 选择联系人/群组 -> 发送消息。

## [Technical Constraints]

### Data Model
- **User**: { id, email, password_hash, nickname, avatar, created_at }
- **Conversation**: { id, type (direct/group), name, created_at }
- **Message**: { id, conversation_id, sender_id, content, type (text/image/audio), file_url, created_at }
- **Membership**: { id, user_id, conversation_id, joined_at }

### State Machine
- **User**: [Created -> Verified -> Active]
- **Message**: [Sending -> Sent -> Delivered -> Read]
- **Conversation**: [Created -> Active -> Archived]

## [Feature Spec]

### 1. 用户认证
- 邮箱注册与登录功能
- 密码重置与账户验证
- 用户资料管理（昵称、头像）

### 2. 核心聊天
- 点对点文字消息
- 高清图片消息（支持压缩与预览）
- 语音消息（支持录制与播放）
- 消息状态显示（发送中、已发送、已送达、已读）

### 3. 群组聊天
- 创建与管理群组（最大50人）
- 群组消息发送与接收
- 群组成员管理

### 4. 消息存储
- 云端永久存储消息历史
- 消息分页加载（默认50条/页），按conversation_id过滤
- 消息搜索功能

### 5. 实时通讯
- WebSocket实时消息推送
- 消息订阅与增量更新（只做增量append，禁止重复订阅）
- 在线状态显示

## 技术实现要点
- 使用 Nuxt4 + Vue3 构建前端
- 后端服务确保消息实时性与可靠性
- 图片与语音消息采用云存储方案
- 实现消息加密与用户数据安全保护
- 优化前端性能，确保流畅的用户体验
- 所有业务表启用RLS，默认"只能访问自己的数据"
- 首屏用服务端拉取，后续靠订阅
- 前端只使用anon key，管理员权限操作走服务端route或server action

## 风险评估报告

### 1. 政策红线风险

#### [高危] 内容合规风险
**风险点描述**：项目涉及用户生成内容（UGC），包括文字、图片和语音消息。
**潜在后果**：如果不接入内容审核机制，可能导致敏感内容传播，面临法律风险和平台下架。
**具体缓解方案**：
- 接入第三方内容审核服务，对用户发送的消息进行实时过滤
- 建立用户举报机制，及时处理违规内容
- 制定明确的社区规范，对违规用户进行处罚

#### [高危] 数据隐私风险
**风险点描述**：项目存储用户个人信息和消息历史，需要符合数据保护法规。
**潜在后果**：数据泄露或滥用可能导致用户隐私受损，面临法律诉讼和声誉损失。
**具体缓解方案**：
- 实现端到端加密，保护消息内容
- 遵循GDPR等数据保护法规，获取用户明确授权
- 定期进行安全审计，确保数据存储安全
- 提供数据导出和删除功能，保障用户数据控制权

### 2. 成本黑洞风险

#### [中等] 云存储成本风险
**风险点描述**：图片和语音消息采用云存储方案，可能面临流量和存储成本失控。
**潜在后果**：用户量增长或恶意上传可能导致存储费用大幅增加，影响项目盈利能力。
**具体缓解方案**：
- 对上传文件大小和类型进行限制
- 实现图片压缩和视频转码，减少存储空间
- 设置存储生命周期管理，对旧消息进行归档或清理
- 监控存储使用情况，设置费用预警机制

#### [中等] 实时通讯成本风险
**风险点描述**：WebSocket实时消息推送需要服务器资源支持，高并发时成本增加。
**潜在后果**：用户量增长或消息高峰期可能导致服务器负载过高，增加运营成本。
**具体缓解方案**：
- 优化WebSocket连接管理，减少无效连接
- 实现消息队列，平滑处理高峰期流量
- 考虑使用第三方实时通讯服务，按需付费

### 3. 工程陷阱风险

#### [高危] 技术架构风险
**风险点描述**：使用Nuxt4 + Vue3构建前端，需要确保技术栈的稳定性和兼容性。
**潜在后果**：框架版本更新可能导致兼容性问题，影响应用稳定性。
**具体缓解方案**：
- 锁定依赖版本，避免自动更新引入问题
- 建立完整的测试体系，确保代码质量
- 预留技术债务清理时间，定期重构优化

#### [高危] 实时通讯性能风险
**风险点描述**：群聊规模限制为50人，实时消息推送可能面临性能挑战。
**潜在后果**：群聊消息量大时可能导致延迟增加，影响用户体验。
**具体缓解方案**：
- 实现消息批量处理，减少网络请求
- 优化前端渲染逻辑，避免消息更新导致的性能问题
- 考虑使用分片技术，将大型群组拆分为多个子群组

#### [中等] 数据一致性风险
**风险点描述**：消息状态管理（发送中、已发送、已送达、已读）需要确保数据一致性。
**潜在后果**：网络不稳定时可能导致消息状态不同步，影响用户体验。
**具体缓解方案**：
- 实现消息确认机制，确保消息可靠送达
- 设计幂等性处理，避免重复消息
- 提供消息同步机制，在网络恢复后自动同步未读消息

#### [提示] 扩展性风险
**风险点描述**：当前设计限制群聊最大50人，未来可能需要扩展。
**潜在后果**：用户需求增长时，现有架构可能无法支持更大规模的群组。
**具体缓解方案**：
- 设计时考虑未来扩展性，预留架构升级空间
- 实现模块化设计，便于后续功能扩展
- 定期评估系统性能，提前规划扩容方案

### 4. 第三方依赖风险

#### [中等] 云服务依赖风险
**风险点描述**：项目依赖云存储和可能的第三方实时通讯服务。
**潜在后果**：第三方服务中断或价格调整可能影响项目运营。
**具体缓解方案**：
- 评估多个云服务提供商，选择可靠性高的服务
- 设计服务降级方案，在第三方服务中断时保证核心功能可用
- 建立监控机制，及时发现并处理服务异常

#### [提示] 安全依赖风险
**风险点描述**：项目依赖的第三方库可能存在安全漏洞。
**潜在后果**：安全漏洞可能被攻击者利用，导致数据泄露或系统被攻击。
**具体缓解方案**：
- 定期更新依赖库，修复已知安全漏洞
- 使用安全扫描工具，检测潜在安全问题
- 建立依赖库管理流程，确保引入的库经过安全评估
